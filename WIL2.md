Controller : 사용자의 요청이 진입하는 지점이며, 요청에 따라 어떤 처리를 할 지 결정해주지만 단지 결정만 해주고 실질적인 처리는 서비스에서 담당한다. 그리고 사용자에게 뷰를 응답으로 보내준다.

Service : 컨트롤러에서 받은 요청을 처리하고, 해당 요청에 따른 작업을 수행하는 역할을 한다.

Repository : 데이터베이스와의 CRUD(Create, Read, Update, Delete) 작업을 처리하고, 해당 작업에 대한 SQL 쿼리를 실행하는 역할을 수행한다.

TDD : 테스트 코드를 먼저 만들고, 테스트를 성공하게 해주는 코드를 작성하는 방식의 개발이다.
TDD를 하는 이유 : 코드의 품질이 높아지며 버그를 사전에 방지할 수 있고, 유지보수성과 개발 생산성을 높일 수 있다.
